# Scientific Calculator Project (Procedural vs OOP)

## 📌 Introduction
The purpose of this project is to develop a Scientific Calculator that performs basic arithmetic operations:
- Addition
- Subtraction
- Multiplication
- Division

This project is implemented using two different programming approaches:
1. Procedural Programming Approach
2. Object-Oriented Programming (OOP) Approach

The main goal is to understand why OOP is more effective for real-world applications compared to procedural programming.

---

# 🟡 Procedural Programming Approach

## 📖 Concept
Procedural programming is a simple programming style where the logic is written in a step-by-step sequence. The entire program flow is handled inside a single function or sequential structure.

## ⚙️ Characteristics
- Code is executed line by line
- No proper use of classes or objects
- Data and logic are combined in one place
- Suitable for small and simple programs

## 👍 Advantages
- Easy to understand for beginners
- Simple implementation
- Faster development for small projects

## 👎 Disadvantages
- No code reusability
- Difficult to manage in large projects
- No data security or hiding
- Code becomes repetitive and unstructured

# 🟢 Object-Oriented Programming (OOP) Approach

## 📖 Concept
OOP is an advanced programming paradigm where the program is designed using classes and objects. It represents real-world entities in a structured way.

In this project, the following OOP concepts are used:
- Encapsulation
- Inheritance

---

## 🔐 Encapsulation
Encapsulation means hiding data and providing controlled access through methods.

In this project:
- Input values (numbers) are kept private
- Access is provided using methods

## 🧬 Inheritance
Inheritance means one class acquires properties and behavior from another class.

In this project:
- A base Calculator class is created
- Different operations (Addition, Subtraction, Multiplication, Division) inherit from the base class

---

## ⚙️ Characteristics
- Modular structure
- Reusable components
- Real-world modeling
- Suitable for large and complex systems

---

## 👍 Advantages
- Code reusability
- Easy maintenance
- Better security through data hiding
- Highly scalable architecture

## 👎 Disadvantages
- Slightly complex for beginners
- Requires more initial design effort

# 📊 Comparison (Procedural vs OOP)

- Procedural programming follows a linear structure, while OOP is modular and object-based.
- Procedural programming has no data protection, while OOP provides encapsulation for security.
- Procedural is suitable for small projects, whereas OOP is ideal for large and real-world applications.
- OOP makes code reuse and maintenance much easier.

---

# 🎯 Conclusion
This project clearly shows that procedural programming is suitable for basic tasks, but OOP is a more powerful, flexible, and scalable approach for professional software development.

In real-world applications, OOP is considered the industry standard.
